# Agent 架构总览

## 当前流水线（6步）

```
用户配置参数
     │
     ▼
┌─────────────────┐
│  01 故事圣经     │  mode: story_bible
│  qwen-max 4096  │  输入: 类型/风格/集数/主角/要求
└────────┬────────┘
         │ storyBible
         ▼
┌─────────────────┐
│  02 角色小传     │  mode: character_bios
│  qwen-max 4096  │  输入: storyBible
└────────┬────────┘
         │ characterBios
         ▼
┌─────────────────┐
│  03 资产登记     │  mode: asset_registry
│  qwen-max 6144  │  输入: storyBible + characterBios + styleConfig
└────────┬────────┘
         │ assetRegistry（人物/场景/道具 + 多角度提示词）
         │ 保存时同步到 assets / asset_prompts
         ▼
┌─────────────────┐
│  04 集数大纲     │  mode: episode_map
│  qwen-max 8192  │  输入: storyBible + characterBios + assetRegistry
└────────┬────────┘
         │ episodeMap（每集一行，含情节/钩子/结尾）
         ▼
┌─────────────────┐
│  05 逐集剧本     │  mode: write_episode  × N集
│  qwen-max 8192  │  输入: storyBible + characterBios + assetRegistry
└────────┬────────┘     + episodeMapText + currentOutline + previousSummaries
         │ episodeContent
         ▼
┌─────────────────┐
│  06 集数摘要     │  mode: summarize_episode  × N集（并行于逐集）
│  qwen-turbo 512 │  输入: episodeContent
└─────────────────┘
         │ summary（3句话，用于向后传递上下文）
```

## 数据依赖关系

- **assetRegistry 是全局命名锚点**：场景名、人物名在 04/05 中被强制约束与此一致
- **previousSummaries**：逐集生成时携带最近 5 集摘要，防止人设漂移
- **styleConfig**：在 03 资产登记时生成带风格标签的提示词，并随 `assetRegistry` 保存，用于同步资产的 `styleConfig`
- **资产登记册同步**：`PUT /api/projects/:id/script` 收到 `assetRegistry` 后解析「类型/名称/提示词-*」，幂等写入 `assets` 和 `asset_prompts`

## 数据存储架构（2026-05-04 更新）

所有项目数据存入 SQLite，单文件 `backend/data/aigc.db`。

```
projects            ← 画布(nodes/edges) + 剧本(storyBible/characterBios/…) 合一存储
                       data TEXT 列存 JSON blob
project_members     ← userId / role(owner|editor|viewer)
project_stats       ← 滚动聚合：agentCallCount / tokenUsed / imageGenCount / estimatedCost
events              ← 追加日志：agent_call / image_gen / stage_complete
assets              ← 资产（含 dna / fields / styleConfig 列）
asset_prompts       ← 多视角提示词（label / prompt / imageUrl / generatedAt）
```

**Token 追踪流程**：
1. 前端调 `/api/script-agent` 时带 `projectId`
2. SSE 流结束时后端读最后一帧的 `usage.prompt_tokens + completion_tokens`
3. 调 `trackAgentCall()` → 写 `events` + 更新 `project_stats`（SQLite 事务）

**协作轮询**：前端每 10 秒 `GET /api/projects/:id/status`，若 `updatedBy !== currentUser` 则弹警告。

## 已知局限

- [ ] 无跨集一致性检查（人物行为是否符合人设全靠 prompt 约束）
- [ ] 无自动 retry（某集失败需手动重新生成）
- [ ] 协作为 Last-write-wins，无 OT/CRDT（多人同时写同一集可能覆盖）
- [ ] 资产登记册同步保留已生成图片，不会覆盖同 label 的旧提示词；如需强制刷新提示词，需要后续增加版本/重同步按钮
