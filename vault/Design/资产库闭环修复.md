# 资产库闭环修复

日期：2026-05-04

## 背景

资产库页面已经具备网格、详情面板、DNA 编辑和多视角提示词，但生成链路存在断点：剧本工作台只保存资产登记册文本，未同步到资产库；资产库生图调用了不存在的接口；Dashboard 字段与后端返回不一致。

## 本轮修复

- `PUT /api/projects/:id/script` 保存 `assetRegistry` 时解析资产登记册，幂等写入 `assets` 和 `asset_prompts`
- 解析字段：`类型`、`名称`、`描述/外形/环境/角色`、`提示词-*`
- 已存在资产按 `projectId + type + name` 更新文本字段，保留已有图片
- 已存在同 label 的提示词不覆盖，避免误删已生成角度图
- `AssetLibrary` 统一调用 `/api/generate-image`，直接使用 `imageUrl` 和 `savedId`
- `/api/dashboard` 返回前端消费的 `totalProjects / totalAssets / totalImages / totalTokens / estimatedCost`
- 资产接口按项目成员权限保护项目资产

## 安全决策

- 根目录历史数据库文件 `backend/*.db`、`*.db-shm`、`*.db-wal` 加入 `.gitignore`
- 项目资产读取需要 `project_members` 中存在成员关系
- 项目资产写入需要 owner/editor，资产创建者也可编辑自己的资产
- 旧 docs 中的 DashScope key 示例改为占位符；历史提交中仍出现过 key 形态值，需要在平台侧轮换/作废

## 后续

- 增加强制重同步按钮：可选择覆盖提示词文本但保留图片
- 资产推送到画布时自动拼接 DNA
- Dashboard 增加最近事件和生成队列
