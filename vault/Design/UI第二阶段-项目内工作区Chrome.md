# UI 第二阶段：项目内工作区 Chrome

日期：2026-05-04

## 目标

把项目内的剧本、生图画布、资产库从各自独立的旧 header，收束为统一的 Studio 工作区壳。先统一项目名、模块切换、状态提示、主题切换和视觉基调，再逐步重做各模块主体。

## 已实现

- 新增 `frontend/src/StudioHeader.tsx`
- 首页左侧 5 个模块从静态导航改为真实 section 切换
- 项目队列：完整项目网格、搜索、重命名、删除、流水线进度
- 剧本流水线：按故事圣经、角色小传、资产登记、集数大纲、逐集剧本展示阶段看板
- 视觉资产：全局资产检索、类型过滤和资产预览
- 视频交付：展示交付候选和当前能力接入状态
- 剧本工作台：顶部切换为固定 Studio Header，保留生成状态、保存提示和错误提示
- 生图画布：顶部切换为浮动 Studio Header，保留保存状态和图片库入口
- 资产库：顶部切换为固定 Studio Header，搜索和类型筛选进入右侧操作区
- 资产库背景和网格密度对齐“影棚中控台 / AI 流水线驾驶舱”方向
- 安全顺手项：移除后端 `changeme-*` 默认密钥，启动时要求显式配置 `JWT_SECRET` 与 `ADMIN_SECRET`

## 设计取舍

- 画布页不做全宽 header，避免减少无限画布可操作区域
- 本阶段不迁移剧本主体表单、资产详情面板和 ReactFlow 节点样式，降低功能回归风险
- 保留浅色主题，但视觉主线优先服务深色 Studio 模式

## 下一步

- 剧本页主体改为“左步骤栏 + 中心文档区 + 右生成控制/版本面板”
- 画布页增加左工具栏和右 Inspector
- 资产库增加批量生成、使用时间线和资产质量状态
